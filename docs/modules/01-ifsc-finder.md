# Module: IFSC Finder

The primary product of Rupeepedia. Lets users look up IFSC codes via cascade filtering (Bank → State → City → Branch) or direct search. Each IFSC code has a dedicated SEO page. Also includes bank pages, state pages, and city pages for SEO scale.

## Frontend Pages

| File | URL Route | Purpose |
|---|---|---|
| `src/pages/HomePage.tsx` | `/` | Landing page with search bar + cascade filters |
| `src/pages/IFSCFinderPage.tsx` | `/ifsc-finder` | Full cascade filter UI |
| `src/pages/IFSCDetailPage.tsx` | `/ifsc/:ifsc` | Individual IFSC code detail page |
| `src/pages/IFSCResultPage.tsx` | `/ifsc-result` | Search results list |
| `src/pages/BankPage.tsx` | `/bank/:slug` | All branches for a bank |
| `src/pages/StatePage.tsx` | `/state/:slug` | All branches in a state |
| `src/pages/CityPage.tsx` | `/city/:slug` | All branches in a city |

## Backend API Endpoints

All mounted under `/api` in `backend/src/routes/api.ts`.

| Method | Path | Returns |
|---|---|---|
| `GET` | `/banks` | All banks (cached) |
| `GET` | `/states` | All states (cached) |
| `GET` | `/ifsc/:ifsc` | Full branch detail for one IFSC code |
| `GET` | `/ifsc/:ifsc/nearby` | Nearby branches (same bank + city) |
| `GET` | `/districts?state_id=&bank_id=` | Districts for a state/bank combo |
| `GET` | `/branches?bank_id=&state_id=&district_id=` | Branch list for cascade filter |
| `GET` | `/banks/:bankSlug/states` | States where a bank has branches |
| `GET` | `/bank/:bankSlug/cities/:stateSlug` | Cities for a bank+state |
| `GET` | `/city/:bankSlug/:stateSlug/:citySlug` | Branches for bank+state+city (paginated) |
| `GET` | `/bank/:slug` | SEO bank page — all branches |
| `GET` | `/state/:slug` | SEO state page — all branches |
| `GET` | `/city/:slug` | SEO city page — all branches |
| `GET` | `/search?q=` | IFSC / branch name search |
| `GET` | `/stats` | DB stats (total branches, banks, states) |

## Database Tables Touched

| Table | Usage |
|---|---|
| `Bank` | Bank dropdown, bank page |
| `Branch` | Core IFSC data |
| `State` | State dropdown, state page |
| `District` | District dropdown |
| `City` | City cascade filter |
| `BankStatePresence` | Bank→State navigation |
| `SearchLog` | Logs search queries |

## URL Structure

```
/ifsc/HDFC0001234          # IFSC detail
/bank/hdfc-bank            # Bank page (uses Bank.slug)
/state/telangana           # State page (uses State.slug)
/city/hyderabad            # City page
/bank/hdfc-bank/telangana/hyderabad   # Bank+State+City branches
```

## Caching

`/api/banks` and `/api/states` are three-tier cached (in-memory → Redis → DB). All other IFSC endpoints hit DB directly with indexes on `ifsc`, `bank_id`, `state_id`, `city_id`.

## SEO

Each IFSC detail page has:
- Unique `<title>` and `<meta description>` via `src/utils/seo.ts`
- Structured data (JSON-LD) for branch information
- Indexed in sitemaps (`sitemap-ifsc-1.xml` through `sitemap-ifsc-4.xml`) generated by the backend

## Known Gaps

- Nearby branches feature is basic (same bank + city match only)
- `Branch.city` is UPPERCASE raw string — display should be title-cased in UI
- Some branches missing `latitude`/`longitude` so map view is partial
